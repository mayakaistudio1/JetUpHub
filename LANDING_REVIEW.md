## ЧАСТЬ 1 — ФАКТИЧЕСКИЕ ОШИБКИ И СМЫСЛОВЫЕ ПРОБЛЕМЫ

### 1.1 — 24x Amplify: отсутствует базовый пример

**Проблема:** На лендинге показан ТОЛЬКО сценарий Dennis Promo ($100 + $100 = $200 → $4,800). Базовый пример ($100 → $2,400) без промо — отсутствует.

**Почему критично:** Пользователь не понимает фундаментальную механику продукта. Dennis Promo — стартовый ускоритель, а не основной продукт. Нельзя объяснять 24x через промо-акцию.

**Исправление:**
- Первый пример: `$100 депозит → торговый счёт MT5 = $2,400`
- Второй пример: `$1,000 депозит → торговый счёт MT5 = $24,000`
- Dennis Promo отдельно: `$100 + $100 бонус = $200 → счёт MT5 на $4,800`

---

### 1.2 — 24x Amplify: «управляемая позиция» — опасная формулировка

**Текущий текст:** _«Amplify структурирует баланс в управляемую позицию с 24× базовой экспозицией — в рамках параметров риска»_

**Проблема:** «Управляемая позиция» звучит как managed account / prop trading / доверительное управление. 24x Amplify — это **множитель депозита**, а не управление чужими деньгами. Это принципиальная юридическая и смысловая разница.

**Было в TRADING_JOURNEY_EDITED.md:** _«24x Amplify multiplies your MT5 deposit by 24. Deposit $100 — trade with $2,400. This is NOT managed exposure and NOT funded trading.»_

**Исправление:**
```
24x Amplify умножает твой депозит на MT5 в 24 раза. $100 → торгуешь на $2,400.
Требуется Community Token для активации. Контроль рисков встроен в структуру.
```

**Убрать:** «управляемая позиция», «в рамках параметров риска» (создаёт ложное впечатление гарантий)

---

### 1.3 — Copy-X: не упомянуто требование Community Token

**Текущий текст:** _«Copy-X передаёт сигналы Sonic и NeoFX напрямую на твой MT5. Верификация через Myfxbook. Автоматическое исполнение — ты мониторишь, система действует.»_

**Проблема:** Community Token (#JETUP для Sonic, #NEO для NeoFX) — обязательное условие активации Copy-X. Без токена подключение невозможно. На лендинге это НЕ упомянуто в описании Copy-X.

**Из базы знаний (JETUP_LANDING_KNOWLEDGE.md, раздел Copy-X):** _«Подключение через Community Token (#JETUP для Sonic, #NEO для NeoFX)»_

**Исправление:**
```
Активируй через Community Token. Copy-X маршрутизирует сигналы Sonic и NeoFX
на твой MT5 автоматически — без ручного исполнения, без пропущенных входов.
Результаты верифицированы на Myfxbook.
```

---

### 1.4 — Community Token: упомянут только 1 раз, абстрактно

**Текущий текст:** _«Токены #JETUP и #NEO открывают доступ к продуктам, партнёрским комиссиям и ранним возможностям внутри всей системы JetUP.»_

**Проблема:** Community Token — это ключ ко всей экосистеме. Он должен быть объяснён конкретно: какой токен к какой стратегии, что он активирует. Сейчас одна абстрактная фраза.

**Исправление — добавить конкретику:**
```
#JETUP → подключение к стратегии Sonic через Copy-X
#NEO → подключение к стратегии NeoFX через Copy-X
Токены активируют Copy-X, 24x Amplify и партнёрские комиссии.
```

---

### 1.5 — Dennis Promo: «бонус +$100» без контекста правил

**Проблема:** Упоминается бонус $100, но не указаны ключевые ограничения:
- Только для новых партнёров
- Только один раз
- Бонус $100 выводится только через 12 месяцев
- При выводе начальных $100 раньше 12 мес — счёт ликвидируется
- Прибыль можно выводить в любое время

**Рекомендация:** Добавить disclaimer или ссылку на полные условия. На лендинге не нужно перечислять все ограничения, но нужна фраза типа: _«Условия и ограничения Dennis Fast Start Promo применяются.»_ — чтобы не создавать впечатление безусловного бонуса.

---

### 1.6 — Lot Commission: $10.50 присутствует, но не в центральном месте

**Текущее состояние:** $10.50 за лот упоминается в развёрнутом тексте, но не виден сразу в метрике продукта.

**Из TRADING_JOURNEY_EDITED.md:** Метрика модуля Product Economics = `$10.50 / per lot` — первый и самый понятный слой дохода.

**Рекомендация:** Сделать $10.50 заметной метрикой, а не просто строкой в описании. Это якорная цифра — она конкретная и понятная.

---

### 1.7 — «Усилить депозит» — двусмысленная формулировка

**Текущий текст кнопки:** _«Усилить депозит»_

**Проблема:** «Усилить» звучит как увеличить доходность или добавить деньги. Точнее: «Умножить депозит ×24» — прямое описание механики.

## ЧАСТЬ 2 — СТРУКТУРНЫЕ И UX-ПРОБЛЕМЫ

### 2.1 — Язык: только русский

**Текущее:** Весь лендинг на русском.

**Из базы знаний:** Целевая аудитория DE/RU/EN. Мария AI работает на трёх языках. Основной рынок — немецкоязычный.

**Рекомендация:** Как минимум, нужен language switcher DE/RU/EN. Немецкий — приоритетный (основной рынок).

---

### 2.2 — Нет Product Section в формате «Ecosystem Hub»

**Из PRODUCT_SECTION_SPEC.md:** Продукты должны быть представлены как interactive hub — 3 ноды (Copy-X, 24x Amplify, IB Portal) вокруг центра JetUP, с «powered by TAG Markets» в центре. 3-слойная архитектура: Overview → Detail Panel → Deep Dive.

**Текущее:** На лендинге продукты показаны в рамках пошагового Journey (Шаг 3 — Модули), но нет визуального hub-диаграммы, нет интерактивного выбора продуктов с detail panel.

**Рекомендация:** Добавить Ecosystem Hub как отдельный визуальный блок (или встроить в Шаг 3). SVG hub-диаграмма с clickable нодами — ключевой визуальный элемент спецификации.

---

### 2.3 — 4 пути выбора (Product / Partner / AI / Full System)

**Из базы знаний (Раздел 7):** Лендинг должен предлагать 4 пути: Product Path, Partner Path, AI Path, Full System Path. Посетитель не просто скроллит — он выбирает путь.

**Текущее:** На лендинге 4 шага последовательного Journey (Фундамент → Путь входа → Модули → Твой ход). Выбор пути в Hero-секции («Как ты хочешь строить доход?» с карточками) реализован, но ведёт ли каждый путь в свой отдельный поток?

**Рекомендация:** Убедиться, что каждый из 4 путей ведёт в свой уникальный поток с соответствующим контентом (Product Path — 4 экрана, Partner Path — 5 экранов, AI Path — 5 экранов, Full System — 6 экранов по спецификации).

**Из базы знаний:** AI-консьерж встроен в интерфейс (НЕ чат-виджет). Появляется мгновенно после выбора пути. Должен быть коротким (1-2 предложения + вопрос), задавать вопросы, направлять к следующему шагу.

**Текущее:** На лендинге есть AI-подсказки (Мария, GPT-4), но неясно, насколько они интерактивны. Формулировки длинные — должны быть 1-2 предложения + конкретный вопрос.

**Примеры исправлений AI-текстов:**

| Было | Должно быть |

| _«AI в JetUP превращает влияние лидера в более доступную, повторяемую и масштабируемую систему. Лидер присутствует одновременно везде.»_ | _«AI масштабирует лидера — не заменяет. Что для тебя приоритетнее: рекрутинг или сопровождение?»_ |
