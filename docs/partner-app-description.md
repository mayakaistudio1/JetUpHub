# Partner Webinar App — Описание функционала

## Общее

Partner App — это Telegram Mini App для партнёров JetUP. Предназначено для управления приглашениями на вебинары, отслеживания результатов и AI-помощи в follow-up коммуникации.

**Доступ:** `/partner-app`
**Авторизация:** автоматическая через Telegram (по Telegram ID). Если партнёр не зарегистрирован — показывается экран "Access Restricted" с предложением зарегистрироваться через Partner Bot.

**Дизайн:** белый минималистичный стиль — светло-серый фон (#F5F5F7), белые карточки с мягкими тенями, синие акценты.

---

## Навигация (нижняя панель)

Внизу экрана — 4 вкладки:

| Иконка | Название | Описание |
|--------|----------|----------|
| Dashboard | **Dashboard** | Главный экран с обзором |
| Video | **Meetings** | Список вебинаров + создание приглашений |
| BarChart | **Statistics** | Статистика по прошедшим мероприятиям |
| Bot | **AI** | AI-ассистент для follow-up сообщений |

---

## 1. Dashboard (Главный экран)

### Приветствие
Вверху — "Welcome back" и имя партнёра.

### Блок "Upcoming Meetings"
Карточки ближайших вебинаров (максимум 3). На каждой карточке:
- Название вебинара
- Дата и время
- Имя спикера
- Мини-статистика: **sent** (сколько раз партнёр создавал приглашение) и **registered** (сколько гостей зарегистрировалось)
- Ссылка "See all" ведёт на вкладку Meetings

При нажатии на карточку — переход на вкладку Meetings.

### Блок статистики (3 карточки)
Три крупные карточки с анимированными счётчиками:

1. **Total Attendees** — общее количество гостей, которые пришли на вебинары этого партнёра
2. **Attendance Rate** — процент конверсии (пришли / зарегистрировались). Показывает подсказку: "High engagement" при >= 40%, иначе "Keep inviting"
3. **Upcoming Scheduled** — количество запланированных мероприятий (ближайшие 7 дней)

### Блок "Past Events"
Список прошедших мероприятий (максимум 3). На каждом:
- Название и дата
- **Reg** — количество зарегистрированных
- **Att** — количество пришедших
- Стрелка вправо — при нажатии переход на вкладку Statistics

Ссылка "See all" — тоже на Statistics.

## 2. Meetings (Вебинары)

### Список вебинаров
Карточки предстоящих вебинаров. На каждой:
- Тип (бейдж, напр. "Trading")
- Язык (напр. "DE")
- Название
- Дата, время
- Спикер (фото + имя)
- Мини-статистика: **sent** / **registered**
- Стрелка вправо для перехода

### Детальная страница вебинара
При нажатии на вебинар открывается детальная карточка:
- Тип, язык, название
- Дата, время
- Спикер (фото + имя)
- Два крупных блока: **Invites Sent** и **Registered**
- Кнопка **"Send Invite"** — запускает процесс создания приглашения
- Ниже — список invite-ссылок партнёра (если есть), с мини-статистикой по каждой (reg/att)
- При нажатии на ссылку — раскрывается список гостей с деталями

### Процесс создания приглашения (3 шага)

#### Шаг 1: Выбор типа приглашения
Две карточки:
- **Personal Invite** — "Отправить напрямую одному человеку" (каналы: Telegram, WhatsApp, E-Mail)
- **Social Share** — "Поделиться в соцсетях" (каналы: Telegram, WhatsApp, Facebook, Instagram)

#### Шаг 2: Выбор стиля сообщения
Три шаблона (с превью текста):
- **Professional** (💼) — формальный стиль. Пример: "Ich möchte Sie herzlich zu unserem exklusiven Webinar einladen..."
- **Friendly** (😊) — дружеский стиль. Пример: "Hey! Ich habe ein spannendes Webinar für dich..."
- **Short & Direct** (⚡) — короткий формат. Пример: "Название — дата, время. Jetzt anmelden: ссылка"

#### Шаг 3: Отправка
Экран с:
- **Invite link** — уникальная ссылка партнёра (с кнопкой копирования)
- **Message preview** — предпросмотр текста в выбранном стиле
- **Send via** — кнопки каналов отправки (зависят от типа на шаге 1):
  - Personal: Telegram, WhatsApp, E-Mail
  - Social: Telegram, WhatsApp, Facebook, Instagram
- Кнопка **"Copy message + link"** — копирует всё сразу
- Примечание: "All registrations via this link are automatically attributed to you."

### Детали гостей (на странице вебинара)
При нажатии на invite-ссылку раскрывается список гостей:
- Имя, email
- Статус: **✓ Attended** (зелёный), **Clicked** (синий) или **No show** (серый)
- Для пришедших: длительность (мин), количество вопросов
- Тексты вопросов, которые гость задавал (если есть)

## 3. Statistics (Статистика)

### Список мероприятий
Карточки всех прошедших мероприятий. На каждой:
- Название
- Дата, время
- Счётчики: **reg** (зарегистрировались) и **att** (пришли)
- Стрелка вправо

### Отчёт по мероприятию
При нажатии на карточку открывается детальный отчёт:

#### Conversion Funnel (воронка конверсии)
Горизонтальные полосы:
- **Registered** (синий) — сколько зарегистрировалось
- **Clicked Zoom** (фиолетовый) — сколько нажали на Zoom-ссылку
- **Attended** (зелёный) — сколько реально присутствовали

Под воронкой — **Conversion rate** в процентах (Attended / Registered).

#### Список гостей
Карточки всех гостей. На каждой:
- Имя, email
- Статус-бейдж:
  - **✓ 58m · 3Q** — присутствовал 58 минут, задал 3 вопроса (зелёный)
  - **Clicked** — нажал на Zoom, но не вошёл (синий)
  - **No show** — не появился (серый)

При нажатии на гостя — раскрывается подробная информация:
- Длительность присутствия (мин)
- Телефон (если есть)
- Нажимал ли на Zoom-ссылку
- **Вопросы, которые задавал** — каждый вопрос отображается отдельной карточкой. Примеры:
  - "Wie hoch ist das Mindestkapital?"
  - "Welche Plattform nutzt ihr?"
  - "Gibt es eine Testphase?"
- Если не задавал вопросов — пометка "No questions asked"

## 4. AI Follow-up Assistant

### Начальный экран
Приветствие: "Hi [Имя]! I'm your AI assistant for recruiting and follow-up. Choose an action or write me directly."

### Быстрые действия (6 кнопок)
Сетка 2x3:

| Кнопка | Что делает |
|--------|-----------|
| **Follow-up: Attendee** | Генерирует follow-up сообщение для гостя, который был на вебинаре |
| **Follow-up: No-Show** | Генерирует сообщение для гостя, который не пришёл |
| **Invite to next event** | Генерирует приглашение на следующий вебинар для тех, кто уже был |
| **Suggest a call** | Генерирует сообщение для назначения личного звонка |
| **Send materials** | Генерирует сообщение для отправки информационных материалов о JetUP |
| **Qualify interest** | Генерирует 3 вопроса для оценки интереса контакта к партнёрству |

### Чат
- Стандартный чат-интерфейс
- Сообщения пользователя — синие (справа)
- Ответы AI — белые карточки (слева)
- Поле ввода внизу с кнопкой отправки
- Индикатор "Thinking..." при ожидании ответа

## 5. Страница приглашения (для гостей)

**URL:** `/invite/:code`
Это страница, которую видит приглашённый гость при переходе по invite-ссылке партнёра.

### Содержимое страницы
- Логотип JetUP
- Бейдж "Persönliche Einladung" (Личное приглашение)
- Баннер мероприятия (если есть)
- Название вебинара
- "Einladung von [Имя партнёра]" — кто пригласил
- Карточка спикера (фото + имя)
- Дата и время (два блока)
- **Countdown-таймер** — обратный отсчёт до начала (Дни / Часы / Минуты / Секунды)
- Блок "Das erwartet dich" — тезисы (highlights) вебинара

### Форма регистрации
Поля:
- **Vollständiger Name** (обязательное)
- **E-Mail-Adresse** (обязательное)
- **Telefon** (необязательное)
- Кнопка **"Kostenlos registrieren"**

### После регистрации
- Сообщение "Registrierung erfolgreich!"
- Кнопка **"Zoom Meeting beitreten"** — открывает Zoom в новом окне
- Примечание: "Klicke auf den Button, um das Zoom-Meeting in einem neuen Tab zu öffnen"

### Обработка ошибок
- Если ссылка не найдена или неактивна — экран "Event nicht verfügbar" с кнопкой "Zurück zur Startseite"

## Модель данных (ключевые сущности)

| Сущность | Описание |
|----------|----------|
| **Partner** | Партнёр JetUP (имя, telegramId, cuNumber, статус) |
| **Schedule Event** | Шаблон вебинара (название, спикер, день, время, тип) |
| **Invite Event** | Конкретное приглашение партнёра (уникальный код, привязка к schedule_event) |
| **Invite Guest** | Зарегистрированный гость (имя, email, телефон, кликнул ли на Zoom) |
| **Zoom Attendance** | Данные участия (длительность, кол-во вопросов, тексты вопросов) |

---

## Воронка (pipeline гостя)

```
Партнёр создаёт invite → Гость открывает ссылку → Регистрируется (имя, email) 
→ Нажимает "Zoom Meeting beitreten" → Присутствует на вебинаре 
→ Данные подтягиваются из Zoom API (длительность, вопросы)
```

Статусы гостя:
1. **Registered** — заполнил форму
2. **Clicked Zoom** — нажал кнопку входа в Zoom
3. **Attended** — реально присутствовал (подтверждено Zoom API)
4. **No show** — зарегистрировался, но не пришёл
