## 1. Обзор системы

Partner System — это экосистема инструментов для партнёров JetUP, позволяющая:

- **Приглашать людей на вебинары** через персонализированные ссылки
- **Квалифицировать кандидатов** с помощью AI на основе DISC-профилирования
- **Отслеживать конверсию** от приглашения до участия в вебинаре
- **Получать AI-рекомендации** по работе с кандидатами

Система работает как **Telegram Mini App** (основной режим) и как **веб-приложение** в браузере (без ограничений доступа).

**Поддерживаемые языки:** Английский (EN), Немецкий (DE), Русский (RU)

## 2. Аутентификация и регистрация

### 2.1 Четыре режима аутентификации

Аутентификация основана на **Telegram ID** партнёра. Система поддерживает четыре способа его получения:

```
┌─────────────────────────────────────────────────────────┐
│                    Вход в Partner App                     │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. Telegram Mini App (автоматический)                   │
│     └─ ID берётся из Telegram WebApp SDK                 │
│     └─ window.Telegram.WebApp.initDataUnsafe.user.id     │
│     └─ Пользователь ничего не вводит                     │
│                                                          │
│  2. Браузер (повторный визит)                            │
│     └─ ID читается из sessionStorage                     │
│     └─ Ключ: 'partnerTelegramId'                        │
│                                                          │
│  3. Браузер (первый визит — ручной ввод)                 │
│     └─ Экран входа с двумя опциями:                      │
│        a) Кнопка "Открыть в Telegram"                    │
│           → Переход: t.me/<bot_username>?start=           │
│           → Dev: @Jetup_partner_test_bot                 │
│           → Prod: @JetUP_Partner_Bot                     │
│        b) Ввод Telegram ID вручную                       │
│           → Если ID найден в базе → вход                 │
│           → Если ID не найден → форма регистрации        │
│                                                          │
│  4. Браузер — доступ без ограничений                     │
│     └─ Приложение открывается без блокировки             │
│     └─ Экран входа предлагает ручной ввод ID или         │
│        ссылку на Telegram                                │
│     └─ Нет сообщения "Access Restricted"                 │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 2.2 Процесс регистрации нового партнёра

Регистрация нового партнёра происходит **внутри Mini App** (экран регистрации), а не через команды бота. Если `telegramId` определён, но партнёр не найден в базе (API возвращает 401), открывается **экран регистрации**:

| Поле | Обязательное | Описание |
|------|-------------|----------|
| Имя | Да | Полное имя партнёра |
| CU-номер | Да | Партнёрский идентификатор в системе |
| Телефон | Нет | Контактный телефон |
| Email | Нет | Электронная почта |

**Процесс:**
1. Партнёр заполняет форму
2. `POST /api/partner-app/register` с заголовком `x-telegram-id`
3. Сервер проверяет уникальность `telegramId`
4. Создаётся запись в таблице `partners`
5. Возвращается профиль + начальная статистика
6. Приложение переходит в состояние `ready` → Dashboard

### 2.3 Полная схема состояний

```
             ┌──────────┐
             │  loading  │ ← Загрузка, определение telegramId
             └────┬─────┘
                  │
           telegramId найден?
          ┌───────┴───────┐
         Да              Нет
          │               │
    ┌─────▼─────┐   ┌────▼──────────────────────────────┐
    │  loading   │   │ Экран входа (браузер)              │
    │  profile   │   │  • Ручной ввод Telegram ID         │
    └─────┬─────┘   │  • Ссылка "Открыть в Telegram"     │
          │          └───────────────────────────────────┘
    Партнёр найден?
    ┌─────┴──────┐
   Да           Нет
    │             │
┌───▼───┐  ┌─────▼──────────┐
│ ready │  │ Registration   │ ← Форма регистрации (внутри Mini App)
│       │  │    Screen      │
└───┬───┘  └────────────────┘
    │
┌───▼──────────────────┐
│  Dashboard / Tabs    │ ← Основное приложение
└──────────────────────┘

Браузер (первый визит):
  Браузер → Экран входа → ручной ввод ID
                        └─ или ссылка в Telegram → Mini App
```

### 2.4 Управление сессией

- **sessionStorage** используется для хранения `telegramId` в браузере (ключ: `partnerTelegramId`)
- Флаг `partnerLoggedOut` контролирует состояние выхода
- При нажатии "Выйти" очищается sessionStorage и показывается экран входа
- Повторный вход восстанавливает сессию

## 3. Экраны Partner App

### 3.1 Навигация — 4 вкладки

| Вкладка | Иконка | Компонент | Назначение |
|---------|--------|-----------|------------|
| Dashboard | LayoutDashboard | `DashboardScreen.tsx` | Обзор показателей и активности |
| Meetings | Video | `WebinarsScreen.tsx` | Список вебинаров и создание приглашений |
| Statistics | BarChart3 | `ReportsScreen.tsx` | Детальные отчёты и воронка конверсий |
| AI | Bot | `AIAssistantScreen.tsx` | AI-ассистент для follow-up |

### 3.2 Dashboard (Панель управления)

Показывает:
- **Приветствие** с именем партнёра ("С возвращением, [Имя]!")
- **KPI-карточки:**
  - Total Attendees — сколько людей посетили вебинары
  - Attendance Rate — процент конверсии из регистрации в посещение
  - Upcoming — количество вебинаров в ближайшие 7 дней
- **Ближайшие встречи** — до 3 вебинаров (только **будущие** события; название, дата, время, кол-во приглашённых/зарегистрированных)
- **Прошедшие события** — до 3 последних с результатами

> **Изменение v1.2.0:** Список "Ближайшие встречи" фильтрует прошедшие события — отображаются только предстоящие вебинары.

### 3.3 Meetings (Вебинары)

**Список вебинаров:**
- Название, спикер, дата, время, язык
- Количество отправленных приглашений и регистраций
- Кнопка "Просмотр и приглашение"

**Детальная страница вебинара:**
- Полная информация о событии
- Статистика: отправлено / зарегистрировано
- Две кнопки: "Social Share" и "AI Invite"
- Список созданных приглашений (invite links)
- Детали по каждому гостю (если есть отчёт)

### 3.4 Statistics (Статистика / Отчёты)

- **Полный список событий** с участием партнёра
- **Воронка конверсии** по выбранному событию:
  - Приглашено → Зарегистрировано → Перешли в Zoom → Посетили
- **Список гостей** с деталями:
  - Имя, email, статус (Attended / Clicked / No-show)
  - Длительность участия, количество вопросов, тексты вопросов

> **Изменение v1.2.0:** Регистрации через Personal AI Invite (`personal_invites.registeredAt`) теперь учитываются в `/api/partner-app/profile` и `/api/partner-app/events` наравне с регистрациями через Social Share (`invite_guests`). Суммарная статистика партнёра отражает оба источника конверсий.

### 3.5 AI Assistant

- Интерактивный чат с AI-рекрутинг-помощником
- Быстрые действия: follow-up для участников, приглашение на новые события, скрипты для звонков

## 4. Система приглашений (Invite System)

### 4.1 Два типа приглашений

```
┌─────────────────────────────────────────────────────────┐
│              Партнёр выбирает тип приглашения             │
├─────────────────────┬───────────────────────────────────┤
│   Social Share      │     Personal AI Invite            │
│                     │                                    │
│   Быстрое создание  │   AI-квалификация + персональная  │
│   ссылки с готовым   │   страница с чат-ботом            │
│   шаблоном          │                                    │
│                     │                                    │
│   Шаблоны:          │   DISC-профилирование:            │
│   • Professional    │   • Определение типа личности     │
│   • Friendly        │   • Подбор стратегии              │
│   • Short           │   • Генерация AI-сообщений        │
│                     │                                    │
│   Каналы:           │   Результат:                      │
│   • Telegram        │   • Уникальная ссылка             │
│   • WhatsApp        │     /personal-invite/:code        │
│   • Facebook        │   • AI-чат на стороне гостя       │
│   • Email           │   • Регистрация внутри чата       │
│   • Instagram       │                                    │
└─────────────────────┴───────────────────────────────────┘
```

### 4.2 Social Share (Обычное приглашение)

1. Партнёр выбирает вебинар
2. Выбирает шаблон сообщения (Professional / Friendly / Short)
3. Система создаёт уникальную ссылку `/invite/:code`
4. Партнёр получает готовое сообщение + ссылку
5. Делится через Telegram / WhatsApp / Email / Facebook / Instagram

**При переходе по ссылке:** Гость видит страницу события с деталями и формой регистрации.

### 5.2 DISC-профилирование

**Что такое DISC:**
Система классификации личности по 4 типам, используемая для персонализации коммуникации.

| Тип | Название | Характеристика | Тон общения AI |
|-----|----------|----------------|----------------|
| **D** | Dominance (Доминирование) | Результативный, решительный, прямой | Прямой, без лишнего, фокус на результатах и масштабе |
| **I** | Influence (Влияние) | Энергичный, вдохновляющий, общительный | Тёплый, вдохновляющий, фокус на новизне и людях |
| **S** | Steadiness (Стабильность) | Спокойный, надёжный, осторожный | Мягкий, уважительный, фокус на доверии и безопасности |
| **C** | Conscientiousness (Добросовестность) | Аналитичный, точный, скептичный | Чёткий, рациональный, фокус на структуре и деталях |

**Логика определения DISC-типа:**
```
Быстрое решение + Деньги/Результаты      → D (Dominance)
Рост бизнеса + Технологии/Инновации       → I (Influence)
Сообщество + Обучение + Нужно доверие     → S (Steadiness)
Аналитик + Скептик                        → C (Conscientiousness)
```

**Логика выбора стратегии:**
```
MLM Leader / Предприниматель              → Authority
Инвестор / Деньги-Результаты              → Opportunity
Друг / Холодный контакт                   → Curiosity
Нужно доверие / Обучение                  → Support
```

### 5.3 Генерация AI-сообщений

Сервер отправляет в GPT-4o-mini промпт, который включает:

1. **Контекст:** Имя партнёра, имя кандидата, тип отношений
2. **Данные вебинара:** Название, спикер, дата/время, ключевые моменты
3. **DISC-тональность:** Инструкции по стилю общения
4. **Стратегия:** Как подать приглашение (Authority/Opportunity/Curiosity/Support)
5. **Язык:** На каком языке генерировать (EN/DE/RU)

**Результат генерации:**
- 2 коротких персональных сообщения
- Quick Replies (быстрые ответы) адаптированные под DISC-тип
- Определённые стратегия и DISC-тип

### 5.4 Превью приглашения (Invite Preview)

После создания приглашения партнёр может нажать "Превью" и увидеть:

- **Имитация чата** — как будут выглядеть AI-сообщения в виде пузырей чата
- **Quick Replies** — кнопки быстрых ответов, которые увидит кандидат
- **Карточка вебинара** — название, дата, время, спикер

### 6.2 AI-чат — как работает

**Системный промпт AI включает:**
- Роль: "Ты — персональный ассистент [Имя партнёра]"
- Задача: пригласить [Имя кандидата] на вебинар
- DISC-тональность: конкретные инструкции по стилю
- Стратегия: как подать приглашение
- Данные вебинара: название, спикер, дата, highlights
- Языковые инструкции: на каком языке отвечать

**Правила AI:**
- Не представляться как AI — выступать от лица партнёра
- Короткие, разговорные сообщения (не корпоративный стиль)
- Мягко подводить к регистрации
- Отвечать на вопросы о вебинаре на основе данных
- Не давать финансовых обещаний

## 8. Telegram-бот для партнёров

Система использует **два отдельных бота** — для разработки и для продакшна:

| Среда | Username | Токен |
|-------|----------|-------|
| **Dev** | `@Jetup_partner_test_bot` | `TELEGRAM_PARTNER_BOT_TOKEN_DEV` |
| **Prod** | `@JetUP_Partner_Bot` | `TELEGRAM_PARTNER_BOT_TOKEN` |

**Логика выбора токена:**
- Если `NODE_ENV=production` → используется прод-бот (`TELEGRAM_PARTNER_BOT_TOKEN`)
- Иначе → используется dev-бот (`TELEGRAM_PARTNER_BOT_TOKEN_DEV`)
- Два бота никогда не пересекаются — каждый слушает только свой webhook URL

**Webhook:** `POST /api/telegram-bot/webhook`

### Команды бота

| Команда | Действие |
|---------|----------|
| `/start` | Открывает Partner Mini App через Web App кнопку |
| `/invite` | Бот отвечает **Web App кнопкой**, открывающей Partner Mini App **на вкладке Вебинаров** — партнёр сразу может выбрать вебинар и создать приглашение |
| `/events` | Показывает статистику по событиям |
| `/report` | Детальный отчёт |
| `/followup` | AI follow-up рекомендации |

> **Изменение v1.2.0 (/invite):** Команда `/invite` больше не создаёт ссылку напрямую в чате. Вместо этого бот отвечает Web App кнопкой, которая открывает Partner Mini App на вкладке Вебинаров — партнёр проходит полный флоу создания приглашения внутри Mini App.

### Уведомления

Бот отправляет партнёру уведомления:
- Когда гость регистрируется через его ссылку
- Когда гость переходит в Zoom

## 11. Promo Admin Panel

### 11.1 Обзор

Отдельная страница `/promo-admin` предназначена для сотрудников, которым нужно верифицировать промо-заявки. Она имеет собственный пароль, независимый от основной `/admin` панели, и предоставляет упрощённый интерфейс только для управления заявками.

### 11.2 Доступ и аутентификация

- **URL:** `/promo-admin`
- **Пароль:** задаётся через переменную окружения `PROMO_ADMIN_PASSWORD` (отдельно от основного `ADMIN_PASSWORD`)
- Аутентификация через `POST /api/promo-admin/login`
- Нет доступа к другим разделам администрирования (логи чатов, спикеры, события, приглашения, партнёры и т.д.)

### 11.3 Интерфейс

Страница показывает только таблицу промо-заявок со следующими полями:

| Поле | Описание |
|------|----------|
| ID | Идентификатор заявки |
| Имя | Имя заявителя |
| Email | Электронная почта |
| CU Number | Партнёрский номер |
| Статус | pending / verified / rejected |
| Дата | Дата подачи заявки |
| Действия | Кнопки "Одобрить" и "Отклонить" |

### 11.4 Действия

**Кнопка "Одобрить" (✓):**
1. Устанавливает статус заявки `verified`
2. Автоматически отправляет подтверждающий email через Resend
3. Обновляет статус в Google Sheet
4. Обновляет список заявок

**Кнопка "Отклонить" (✗):**
1. Устанавливает статус заявки `rejected`
2. Обновляет список заявок

### 11.5 Мобильная поддержка

Интерфейс адаптирован для мобильных устройств — сотрудники могут верифицировать заявки со смартфона.

### 11.6 Связь с основным Admin Panel

Вкладка Promo в основной `/admin` панели продолжает работать параллельно. Promo Admin Panel — дополнительный упрощённый интерфейс, не заменяющий основной.
