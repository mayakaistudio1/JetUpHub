# Архитектура режима презентации

## Общая структура

Презентация — fullscreen overlay (z-index: 1000) поверх всего.
Компонент: `client/src/pages/partner/PresentationOverlay.tsx`
Стили: `client/src/pdh-v3.css`

Ключевая особенность: **общий стейт сообщений**. Messages живут в `PartnerDigitalHub` и передаются как пропсы в `ChatOverlay` и `PresentationOverlay`. Любое сообщение, добавленное из презентации, автоматически видно в чате.

## Навигация

### Кнопки (внизу экрана)
- **← Назад**: предыдущий слайд (disabled на первом)
- **Далее →**: следующий слайд (акцентный цвет текущего слайда)
- **Индикатор чата**: посередине, показывает кол-во сообщений, открывает чат по нажатию

### TOC (оглавление)
- Кнопка-список (☰) в правом верхнем углу
- Открывает drawer справа (260px) со списком всех 12 слайдов
- Текущий слайд подсвечен акцентным цветом
- Слайды с заданным вопросом отмечены галочкой ✓
- Нажатие на слайд = переход + закрытие drawer

### Progress bar
Тонкая полоска (3px) в самом верху. Ширина = `(current + 1) / 12 * 100%`. Цвет = акцент текущего слайда.

### Анимация переходов
```
Вход:   x: ±280px, opacity: 0, scale: 0.95
Центр:  x: 0, opacity: 1, scale: 1
Выход:  x: ∓280px, opacity: 0, scale: 0.95
Длительность: 0.35s, easeInOut
```

## Предложенные вопросы → Основной чат

На каждом слайде внизу — кнопка с предложенным вопросом (например, «Как работает безопасность?»).

### Механика
1. Пользователь нажимает кнопку
2. Вопрос добавляется в общий массив `messages` как `sender: "user"`
3. Через 500ms AI-ответ (`detailedAnswer`) добавляется как `sender: "ai"`
4. Кнопка меняется на «✓ Отправлено в чат» (disabled, зелёный)
5. Toast-уведомление «Добавлено в чат» появляется на 2 секунды
6. Слайд отмечается в TOC галочкой
7. Повторное нажатие невозможно (отслеживается через `askedSlides: Set<number>`)

### Код
```
handleAskQuestion() {
  if (askedSlides.has(current)) return;
  askedSlides.add(current);
  addMessage({ text: slide.suggestedQuestion, sender: "user" });
  setTimeout(() => {
    addMessage({ text: slide.detailedAnswer, sender: "ai" });
  }, 500);
}
```

## Чат в презентации (свайп вниз)

### Два способа открыть
1. **Свайп вниз** по области слайда (drag gesture, порог 80px)
2. **Нажатие на индикатор чата** внизу экрана (между кнопками навигации)

### Что показывает
- Полную историю основного чата (тот же массив `messages`)
- Включая все вопросы, заданные со слайдов
- Можно писать свободно (input + send)
- Ответы AI также добавляются в общий массив

### Закрытие
- **Свайп вверх** (drag gesture, порог 100px)
- **Кнопка X**
- **Нажатие на backdrop**

### Визуал
- Bottom sheet 75vh
- Handle (полоска) сверху для drag
- Тёмный фон #0F0A1A
- Аватар Dennis AI + статус Online

- ⚫ **Начать по моей ссылке** (glass)

---

## Возврат в чат после презентации

При возврате (X, «Вернуться в чат», или CTA):
1. `PartnerDigitalHub` ставит `presentationWatched = true`
2. `ChatOverlay` через 600ms добавляет follow-up:
   > «Что из этого откликнулось больше всего? Давай разберём твой кейс лично.»
3. Все сообщения из презентации уже в чате

## Shared State (архитектура данных)

```
PartnerDigitalHub
├── messages: SharedMessage[]     ← единый массив
├── addMessage(msg) → id          ← единая функция
│
├── ChatOverlay
│   ├── messages (prop)           ← читает
│   ├── addMessage (prop)         ← пишет
│   └── handleUserMessage()       ← добавляет user + ai
│
└── PresentationOverlay
    ├── messages (prop)           ← читает (для чата)
    ├── addMessage (prop)         ← пишет (вопросы со слайдов)
    ├── handleAskQuestion()       ← suggestedQuestion → messages
    └── handleChatSend()          ← свободный ввод → messages

SharedMessage {
  id: number
  text: string
  sender: "ai" | "user"
  type?: "presentation_trigger"
}
```
